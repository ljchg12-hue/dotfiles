## 사용 방법

### 1. 일반 질문 (자동 검증)

단순히 질문만 하면 자동으로 검증합니다:

```
질문: Claude API의 최대 컨텍스트는?

답변 형식:
🟢 **확인됨 (Verified)** | 검증 시간: 6분

## 핵심 답변
[검증된 답변]

## 📄 출처 상세
[3개 이상 출처]
...
```

### 2. 빠른 확인

긴급하게 확인이 필요한 경우:

```
빠르게: Python 최신 버전

→ 2-3분 내 간단 검증
```

### 3. 학술급 검증

논문이나 연구에 사용할 경우:

```
학술급으로: Transformer 아키텍처 발전 (2020-2024)

→ 15-20분 심층 검증
→ IEEE/APA 인용 형식
```

### 4. 비교 분석

여러 옵션 비교:

```
Claude Sonnet 4 vs GPT-4o 비교:
- 코드 생성 정확도
- 가격 (한국 원화)
- 컨텍스트 크기

→ 교차 검증 비교표
→ 한국 맥락 자동 반영
```

## 신뢰도 라벨 이해하기

### 🟢 확인됨 (Verified)
- **의미**: 3개 이상 독립 출처에서 완전히 일치
- **신뢰도**: 매우 높음 (95%+)
- **사용**: 공식 문서, 중요 의사결정에 활용 가능

### 🟡 부분 검증 (Partial)
- **의미**: 2개 출처 또는 일부 정보 불일치
- **신뢰도**: 중간 (80-90%)
- **사용**: 참고용, 중요 결정 시 재확인 필요

### 🟠 제한적 검증 (Limited)
- **의미**: 1개 출처만 확인 또는 신뢰도 낮음
- **신뢰도**: 낮음 (60-75%)
- **사용**: 참고만, 반드시 원본 확인 필요

### 🔴 검증 불가 (Unverifiable)
- **의미**: 신뢰 가능한 출처 없음
- **신뢰도**: 불명
- **사용**: 검증 불가 항목으로 처리

## 문제 해결

### Q: 검증 시간이 너무 오래 걸려요

**A**: 검증 수준 하향 또는 질문 세분화
```
"학술급" → "표준 수준"
"AI 전반에 대해" → "GPT-4 function calling"
```

### Q: 출처가 부족하다고 나와요

**A**: 검색어 변경 또는 영어로 재시도
```
"Claude Code SSH" → "Model Context Protocol SSH server"
한글 검색 → 영문 검색
```

### Q: 출처 간 정보가 다릅니다

**A**: 최신 출처 우선 또는 공식 출처 우선
```
"가장 최근 업데이트된 출처 기준으로"
"공식 발표 기준으로 재검증"
```

### Q: 한국 정보가 부족해요

**A**: 영어 출처 + 한국 환산 요청
```
"영어 출처로 검증 후 한국 맥락으로 변환"
```
